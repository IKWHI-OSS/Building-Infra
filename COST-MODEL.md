# COST-MODEL — 과금 구조 복기 장치

> 목적: 지금은 "LLM 호출당 과금"만이지만, 아래 셋 중 하나라도 도입하면 **과금 구조가 달라진다.** 그 경계를 넘기 전에 이 표를 다시 본다.
> 자동 장치: 노트북 첫 부분의 `cost_notice()`가 실행 시 env를 감지해 해당 항목을 경고한다.

## 현재 (PoC) — 호출당 과금만

| 항목 | 과금 | 비고 |
|---|---|---|
| LLM API (OFFER 분석·요약, s4 judge) | **호출(토큰) 단위** | 동작 있을 때만. 유휴 상태 0. 시나리오 A ≈ 3회 호출, B는 LLM 미호출 |
| 웹검색 MCP (키리스 DuckDuckGo) | **무료** | 키 없음. 단 rate-limit/차단 가능 |
| 로컬 실행 (노트북·MCP 서버 서브프로세스) | **무료** | 내 맥에서 도는 프로세스 |

→ 한 줄: **동작(호출)이 있어야만 비용. 가만히 두면 추가 비용 없음.**

## 경계를 넘으면 — 과금 구조가 바뀐다

| 진입하는 것 | 어떻게 과금이 바뀌나 | 감지 트리거(가드가 보는 것) |
|---|---|---|
| **① 유료 검색 API** (Tavily, Serper, Brave 등) | 검색 **호출당 과금**(무료 검색 → 유료). 키리스 DDG와 다른 비용축 추가 | `TAVILY_API_KEY` / `SERPER_API_KEY` 등 env 존재 |
| **② 상시 떠 있는 서버** (MCP 서버·API를 서비스로 배포) | **시간당 호스팅비**(유휴여도 과금). "호출당" 모델에서 "상시 과금"으로 전환 | 배포 단계 진입(노트북 밖) — 가드가 자동 감지 불가, 이 표로 수동 복기 |
| **③ LangSmith 유료플랜** (트레이싱) | **트레이스 전송량 단위 과금**. 무료 한도 초과분 | `LANGSMITH_API_KEY` / `LANGCHAIN_TRACING_V2=true` env |

## 점검 습관
- 새 `*_API_KEY`를 `.env`에 추가하기 전에: 그 서비스의 과금 페이지를 한 번 확인.
- "서버를 띄워 두자"는 결정 전에: 유휴 호스팅비가 호출당 비용보다 합리적인지 따져본다.
- 트레이싱을 켜기 전에: 무료 한도와 초과 단가를 본다.
